SNHU CS 210 Portfolio Repository
Corner Grocer Project

Project Summary
The Corner Grocer Project was designed to analyze a grocery store’s daily sales by 
reading item data from an input file and displaying how many times each item appeared. 
The program alloes the user to search for an individual item's purchase frequency, 
display the frequency of all items, and generate a histogram for a visual 
representation of the data. This project demonstrates the use of file input and 
output, maps, functions, and user interaction in C++.

What I Believe I Did Well
One of the strongest parts of this project was organizing the code into seperate 
functions in order to keep it readable and maintainable. I also implemented input 
validation and created a menu driven interface that allows users to easily navigate 
the program. Using a map to store and retrieve the item frequencies not only made the 
program efficient but also easy to expand.

Areas for Improvement
If I continued to develop this project, I would improve the UI by making the menus 
more user friendly and allow users to process different input files without modifying 
the source code. I would also add more error handling and comments in order to make 
the program easier for other developers to understand.

Challenges and How I Overcame Them
The most challenging part of the project for me was working with file input and using 
a map to correctly count the occurrence of each grocery item. I overcame these 
challenges by testing the program with different inputs and breaking the problem into 
smaller functions.

Transferable Skills
This project strengthened my understanding of C++ programming, object-oriented design, 
file input and output, data structures, modular programming, and debugging. These 
skills will be valuable in future software development projects and other computer 
science courses.   

Maintainability and Readability
I improved the maintainability of the program by seperating tasks into individual functions, using descriptive variable names, following consistent formatting, and organizing the code into header and source files. These practices make the code easier to read, update, and reuse in future projects.
